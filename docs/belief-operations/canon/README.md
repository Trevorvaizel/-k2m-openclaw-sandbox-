# K2M Belief Operations Canon

This folder is the primary doctrine source for belief operations work.

Use it to answer:

- what belief architecture is
- how belief moves through a market
- how topology shapes spread
- how trust, resistance, and defining moments change conversion

Do not use the operating docs or templates to settle doctrinal questions if the canon already covers them.

## Source Precedence

1. [../../k2m-layer4-product-brief.md](../../k2m-layer4-product-brief.md) for product truth
2. the canon in this folder for belief doctrine
3. the operating docs in [../](../) for implementation architecture
4. the templates in [../templates/](../templates/) for output shapes

If two sources conflict:

- product claims defer to the product brief
- belief mechanics defer to the canon
- workflow or implementation details defer to the operating docs

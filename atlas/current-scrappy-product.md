# Current Scrappy Product

## Definition

The current scrappy K2M product is an 8-week Discord-based critical-thinking cohort run through KIRA, a student-facing Discord bot, plus a landing page, Google Sheets context engine, PostgreSQL runtime store, facilitator dashboard, and parent-update paths.

## Pattern

The visible product is practical and incomplete:

- students enroll through a landing page
- Discord becomes the daily learning environment
- KIRA handles onboarding, prompts, agent commands, habit tracking, safety checks, and progression
- students move through weekly channels, reflections, and agent unlocks
- the product culminates in a thinking artifact
- Trevor remains the manual facilitator and escalation owner

This is enough to embody the kernel, but not enough to be treated as the final product architecture.

## Diagnostic Use

Ask:

- What is live in Discord or the landing flow right now?
- What is manual but operationally real?
- What is target architecture but not current-state truth?
- Which product gaps block the kernel from being experienced?

## Trap

The trap is overpolishing the current vessel before learning what the kernel wants to become. The current product is valuable because it gives the kernel a body and generates reality contact.

# Session Prompt Engine

## Definition

The session prompt engine is the daily practice layer that turns the cohort philosophy into repeated behavior. It binds weekly themes, habit reinforcement, prompt cadence, peer visibility moments, and Trevor's live sessions into one rhythm.

## Pattern

Its first-pass shape is:

- Monday-Thursday async prompts keep the cohort moving daily
- Friday reflections handle weekly consolidation separately
- prompts reinforce the active habit and the week's identity shift
- Wednesday moments create peer visibility without ranking
- live session scripts provide human ceremony and reinforcement where the async layer would be too thin
- prompts and NotebookLM scripts pull from the same approved example logic that the agents use, instead of inventing their own example universe

This is how the product turns abstract doctrine into repeated contact with the learner.

## Diagnostic Use

Ask:

- How does the product create daily rhythm rather than isolated tool use?
- Where is habit reinforcement encoded in the content system?
- Which moments are private reflection, which are public visibility, and why?

## Trap

The trap is treating the prompt library like copy inventory. It is actually the behavior engine for the cohort.

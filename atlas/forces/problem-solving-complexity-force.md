# Complexity Force

> Context decides method.

## Definition
The complexity force is the requirement to distinguish clear, complicated, complex, and chaotic problems before choosing how to intervene.

## Pattern
- some terrains reward standardization
- others reward expertise and analysis
- others demand experimentation because the system is adaptive and path-dependent
- certainty can signal competence in one terrain and delusion in another

## Source-Grounded Example
The manifesto uses the clear/complicated/complex/chaotic distinction to show why a favorite method becomes dangerous when applied to the wrong species of problem.

## Diagnostic / Use
Ask:
- What kind of terrain is this?
- Is analysis enough, or is experimentation mandatory?
- Are local best practices helping or becoming a liability?
- Does this system stabilize first or diagnose first?

## Trap or Limitation
Using complexity as an excuse for vagueness. Naming the terrain should sharpen method choice, not erase accountability.

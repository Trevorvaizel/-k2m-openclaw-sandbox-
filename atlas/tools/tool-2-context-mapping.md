# Tool 2: Context Mapping

> *Know WHEN, not just WHAT. Map the boundaries where rules break.*

## The Process
1. Take any rule/technique
2. Map the contexts where it **works**
3. Map the contexts where it **fails**
4. Map the **boundary** between them
5. Name the contexts

## The Result
You become context-aware (Level 2 of [[context-spectrum|Force 6]]). You stop applying rules blindly. You start matching approach to situation.

## Force Connections
- Direct application of [[context-spectrum|Force 6]]
- Reveals [[emergent-complexity|Force 5]]: contexts are where emergence happens
- Feeds [[attention-economics|Force 8]]: knowing contexts tells you where to look

## Cross-Domain Example
**AI:** Deep learning works for perceptual tasks with lots of data. It fails for causal reasoning with limited data. The boundary: the structure of the problem space. Map that boundary and you know when to reach for which tool.

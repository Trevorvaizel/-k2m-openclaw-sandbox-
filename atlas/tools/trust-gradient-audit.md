# Trust Gradient Audit

> Correct diagnosis without adoption architecture is unused intelligence.

## Purpose
Use the trust gradient audit to identify what trust layer a change effort requires, what layer is actually built, and what proof artifact closes the next gap.

## Process
Ask:
1. What layer of trust does this intervention require?
2. What layer has actually been built?
3. What proof artifact closes the next gap?

Then match the intervention to the next trust layer rather than arguing with a receiving system that cannot yet metabolize the diagnosis.

## Result
- implementation resistance becomes more legible
- evidence gets paired with adoption design
- teams stop mistaking disbelief for stupidity

## When To Use
Use this when a diagnosis is analytically strong but repeatedly fails to travel.

## Failure Mode
Treating trust as a communications garnish instead of part of the mechanism.

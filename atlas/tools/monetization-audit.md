# Monetization Audit

> Pricing and packaging reveal what the product really believes value is.

## Purpose
Use the monetization audit to expose the value unit, customer subsidy pattern, scaling cost, and packaging truth underneath the business model.

## Process
Ask:
1. What value unit is really being sold?
2. Which customer is subsidizing which use case?
3. Where do costs scale nonlinearly?
4. Where does packaging obscure true value?
5. Does the pricing model preserve or damage trust?

## Result
- fuzzy value assumptions become visible
- pricing stops being postponed as a post-product decision
- AI cost and value structures become part of product architecture

## When To Use
Use this when usage is rising but margin, customer quality, or buyer trust is weakening.

## Failure Mode
Treating monetization as extraction instead of truth alignment.

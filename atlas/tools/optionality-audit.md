# Optionality Audit

> Preserve the ability to learn before you preserve the appearance of certainty.

## Purpose
Use the optionality audit to check whether a team has made irreversible commitments before buying enough information.

## Process
Ask:
1. What commitments have we already made?
2. Which of them are reversible?
3. What experiment would buy the next useful information cheaply?
4. What option disappears if we lock this in now?
5. What downside remains unsurvivable?

## Result
- teams can see where they hardened too early
- cheap learning gets prioritized over expensive certainty theater
- option-killing commitments become visible

## When To Use
Use this when a roadmap, budget, architecture, or narrative is crystallizing faster than the evidence.

## Failure Mode
Confusing optionality with drift. The audit should sharpen search discipline, not excuse indecision.

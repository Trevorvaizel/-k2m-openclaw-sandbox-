# Constraint Board

> Keep the current limiting factor explicit or local optimization will quietly retake the system.

## Purpose
Use a constraint board to track the real limiter, the evidence that it is the limiter, the move to exploit it, the move to protect it, and the signal that the constraint has shifted.

## Process
Maintain five fields:
1. current limiting factor
2. evidence it is truly the constraint
3. exploitation move
4. protection move
5. signal the constraint has shifted

Review it whenever throughput claims improve without obvious system change.

## Result
- bottleneck drift becomes more visible
- teams stop celebrating local motion as system progress
- improvement attention stays disciplined

## When To Use
Use this when multiple teams are improving in parallel but the system still feels slow, blocked, or politically jammed.

## Failure Mode
Locking onto the first named bottleneck and never updating the board when the system changes.

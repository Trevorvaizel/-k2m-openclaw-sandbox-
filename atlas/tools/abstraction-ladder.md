# Abstraction Ladder

> Move the problem up and down levels deliberately before choosing the intervention.

## Purpose
Use the abstraction ladder to stop overfitting your intelligence to one layer of reality.

## Process
Move the problem through these levels:
1. event
2. process
3. system
4. paradigm

At each level, ask what changes, what stays hidden, and which interventions become newly available.

## Result
- local symptoms stop masquerading as whole-system diagnosis
- paradigm errors become visible earlier
- teams can choose the level of attack more deliberately

## When To Use
Use this when the same problem keeps returning despite competent local fixes.

## Failure Mode
Climbing abstractions without returning to action. The ladder exists to improve intervention, not to reward elevation.

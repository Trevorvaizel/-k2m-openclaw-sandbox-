# Friction Symmetry Audit

> Compare opposite paths before declaring the interface fair.

## Purpose
Use the friction symmetry audit to test whether opposite choices carry proportionate cost.

## Process
Compare:
1. enter / exit
2. accept / decline
3. buy / compare
4. continue / stop
5. personalize / reset
6. consent / refuse
7. subscribe / cancel

Then ask whether the asymmetry is justified or exploitative.

## Result
- dark-pattern logic becomes more legible
- protective friction can be separated from coercive friction
- teams can defend or redesign their choice architecture with clearer reasoning

## When To Use
Use this when a system feels sticky, manipulative, or suspiciously one-way.

## Failure Mode
Assuming perfect symmetry is always the goal. Some asymmetry is legitimate; hidden asymmetry is the danger.

# KIRA As Belief Interface

## Definition

KIRA is the current product interface where AI-assisted thinking becomes a repeatable belief-revision experience. It does not merely answer student questions; it stages the conditions for framing, divergence, challenge, synthesis, and proof.

## Pattern

KIRA expresses the product kernel through:

- onboarding that reduces uncertainty before performance pressure
- agent commands that separate thinking moves
- habit tracking that turns identity change into repeated evidence
- safety and escalation checks that protect the learner while the model shifts
- artifact assembly that makes changed thinking visible

KIRA is a belief interface because it touches the learner at the moment their self-story is active.

## Diagnostic Use

Ask:

- Does KIRA surface the student's model, or only provide output?
- Does the interaction protect agency and authorship?
- Does the agent flow leave a proof object behind?
- Where does KIRA need Trevor's human judgment because the belief state is too delicate?

## Trap

The trap is designing KIRA as a chatbot instead of a thinking interface. A chatbot optimizes response quality. KIRA should optimize belief-safe agency and visible thinking.
